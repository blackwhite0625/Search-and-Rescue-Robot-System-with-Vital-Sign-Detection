# ROS 具生命跡象偵測之搜救機器人系統

## 作品說明

本專案是一套結合 ROS、影像辨識、生命跡象推估、LiDAR 避障與網頁監控介面的搜救機器人系統。作品目標是在災害、倒塌建物、低可視度或人員不易進入的環境中，讓機器人能先行巡邏、尋找疑似傷患，並將影像、位置、風險狀態與生命跡象資訊回傳給救援人員，協助現場快速判斷優先救援目標。

系統以 Raspberry Pi / Linux 環境為主要運行平台，搭配攝影機、LiDAR、馬達控制板、舵機、喇叭與麥克風模組。前端以 Flask 網頁介面提供即時影像、手動控制、自動巡邏狀態、AI 偵測結果、LiDAR 距離、警報紀錄與生命徵象資訊，使用者可透過手機或電腦瀏覽器進行監看與操作。

## 核心功能

- **人體與倒地偵測**：使用 YOLOv8 Pose ONNX 模型進行人體骨架偵測，依據肩膀、髖部、身體比例與姿態變化判斷是否疑似倒地。
- **生命跡象推估**：透過 rPPG 分析臉部影像綠色通道變化，推估心率與呼吸頻率，並融合眨眼、微動、意識狀態等資訊形成生命徵象分數與可信度。
- **自動搜救流程**：自動模式下會巡邏環境，偵測到倒地者後靠近至安全距離，播放警報並主動詢問是否需要協助。
- **傷患記憶與回報**：系統會記錄已通報傷患的位置與追蹤資訊，避免同一名傷患被重複通報；完成回報後會離場並繼續巡邏。
- **LiDAR 避障與 SLAM 地圖**：整合 LiDAR 距離資料與 ROS/SLAM 地圖資訊，用於巡邏、避障、定位與傷患位置標記。
- **Telegram 通知**：偵測到高風險事件時，可將警報、生命徵象、信心度與現場影像傳送至 Telegram。
- **手動 / 自動 / Demo 模式**：支援手動遙控、自動巡邏搜救，以及展示用 Demo 流程。

## 搜救流程

1. 機器人進入自動模式後開始巡邏。
2. 攝影機影像送入 YOLO Pose 進行人體骨架偵測。
3. 若判定疑似倒地，系統會結合風險分數、生命跡象與 LiDAR 距離進入鎖定流程。
4. 機器人靠近傷患但保留安全距離，避免撞擊或壓迫傷患。
5. 系統播放警報，並透過語音詢問傷患是否需要幫助。
6. 若傷患有回應，紀錄互動結果並回報；若無回應，系統再次發出警報並提高風險等級。
7. 傷患資料被寫入已通報記憶，避免重複觸發。
8. 機器人離開現場，回到巡邏流程，繼續搜尋附近是否還有其他傷患。

## 生命跡象偵測設計

本系統的生命跡象偵測屬於非接觸式輔助判斷，不是醫療級量測。rPPG 會從臉部區域擷取綠色通道平均亮度，累積成時間序列後進行濾波與頻譜分析，估算心率與呼吸頻率。系統也會參考眨眼、微動、姿態與語音互動狀態，產生綜合生命徵象分數。

由於 rPPG 容易受光線、鏡頭晃動、臉部遮擋、距離過遠與 FPS 不穩影響，因此系統會同步顯示可信度。若臉部不可見或訊號不足，生命跡象會顯示為建立中或未知，避免把不穩定數據誤判為可靠結果。

## 系統架構

- `app.py`：Flask 主程式、模式切換、任務流程、API 與背景執行緒。
- `detector.py`：YOLO 偵測、人體骨架分析、倒地判斷與畫面標註。
- `rppg.py`：非接觸式心率與呼吸推估。
- `vital_signs.py`：生命徵象融合、分數、狀態與可信度計算。
- `mission_controller.py`：搜救任務狀態機。
- `camera.py`：攝影機讀取、影像快取與 MJPEG 串流。
- `motor.py` / `servo.py`：馬達與舵機控制。
- `ros_bridge.py`：ROS、LiDAR、SLAM 與定位資料整合。
- `victim_memory.py`：已通報傷患記憶與去重。
- `templates/`、`static/`：網頁監控與操作介面。
- `model/`：YOLO ONNX 模型。

## 執行方式

安裝 Python 套件：

```bash
pip install -r requirements.txt
```

若需要 Telegram 通知，請先設定環境變數：

```bash
export TELEGRAM_BOT_TOKEN="你的 Telegram Bot Token"
export TELEGRAM_CHAT_ID="你的 Chat ID"
```

啟動系統：

```bash
python3 app.py
```

預設網頁介面：

```text
http://<機器人IP>:5001
```

## 作品特色

本作品不只停留在「偵測到人」或「偵測到倒地」的階段，而是加入完整的搜救任務邏輯。系統會主動靠近傷患、保持安全距離、詢問是否需要幫助、依照回應決定警報與通報策略，並記住已通報目標後離場續巡。這讓機器人更接近實際搜救場景中需要的自主行為，而不是單純的影像辨識展示。
